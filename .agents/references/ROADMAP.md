# Roadmap

> Generated by Sauron from the discovery interview. Treat as the source of truth for this project.

## Milestone 1: Project Scaffolding & Core Infrastructure
Establish the repo layout, dependency management, config, logging, and cache manager that every later milestone builds on.

  - **Set up repo structure and dependency management** _(claude)_ — Create the app/core, app/workers, app/ui, app/models, app/cache, app/setup, tests/, and installer/ folders. Set up pyproject.toml with dependencies: PySide6, audio-separator, resemble-enhance, pedalboard, pyloudnorm, soundfile, imageio-ffmpeg, torch, pytest, pytest-qt.
  - **Build AppConfig, logging, and CacheManager** _(claude)_ — Central config singleton, rotating file logger writing to the cache dir, and a CacheManager class that owns %LOCALAPPDATA%\MusicMasteryEnhancer\cache with subfolders for stems, renders, presets, and logs.
  - **Write scaffolding smoke tests** _(jules)_ — Verify folder creation, AppConfig defaults, and CacheManager path behavior with pytest.

## Milestone 2: Audio Ingestion & Format Conversion
Load a Suno .wav or .mp3 and normalize it to 44.1kHz/24-bit WAV for the rest of the pipeline.

  - **Implement ingestion module** _(claude)_ — app/core/ingestion.py: accept .wav/.mp3 input, decode via soundfile/ffmpeg, resample/re-bit-depth to 44.1kHz 24-bit WAV, write into the track's cache folder.
  - **Write ingestion tests** _(jules)_ — Test conversion correctness across mono/stereo wav, mp3, and unsupported formats.

## Milestone 3: Stem Separation Engine
Split the normalized track into vocal and instrumental stems using audio-separator with a BS-RoFormer model, plus the first-run model download backend.

  - **Integrate audio-separator stem separation** _(claude)_ — app/core/separation.py wrapping audio-separator with the BS-RoFormer checkpoint, GPU/CPU auto-detection, and caching of stem outputs.
  - **Build model download manager backend** _(claude)_ — app/setup/model_downloader.py: downloads BS-RoFormer and resemble-enhance weights with progress callbacks and checksum verification.
  - **Test separation and model download logic** _(jules)_ — Test stem separation caching behavior and downloader error/progress handling with mocks.

## Milestone 4: Vocal Cleaning Pipeline
Neural denoise/enhance the vocal stem via resemble-enhance (cached) and apply the adjustable Pedalboard DSP chain (HPF, LPF, 4kHz notch, de-esser).

  - **Integrate resemble-enhance with per-stem caching** _(claude)_ — app/core/vocal_chain.py neural stage: runs resemble-enhance denoise/enhance on the vocal stem, cached by settings hash so DSP-only re-renders skip it.
  - **Build vocal Pedalboard DSP chain** _(claude)_ — app/core/vocal_chain.py DSP stage: HPF 80Hz, LPF 14.5kHz, adjustable 4kHz peak notch, and a de-esser (sidechain compression on 5-8kHz band).
  - **Test vocal DSP chain correctness** _(jules)_ — Verify filter cutoffs, notch depth, de-esser behavior, and blend interpolation with synthetic signals.

## Milestone 5: Instrumental Cleaning Pipeline
Apply the same neural+DSP pattern to the instrumental stem with lighter default settings and full granular controls.

  - **Build instrumental neural + DSP chain** _(claude)_ — app/core/instrumental_chain.py mirroring vocal_chain.py's structure with lighter default resemble-enhance settings and its own DSP parameters.
  - **Test instrumental chain** _(jules)_ — Verify neural caching and DSP behavior for the instrumental path, and that shared logic with vocal_chain isn't duplicated incorrectly.

## Milestone 6: Remix, Mastering & Export Engine
Blend processed vocal and instrumental with gain controls, apply the final LUFS-normalized limiter, and export a 24-bit WAV.

  - **Build remix and mastering stage** _(claude)_ — app/core/remix_master.py: mix vocal+instrumental with gain controls, measure/normalize LUFS, apply true-peak limiter, export 24-bit WAV.
  - **Test remix/mastering/export accuracy** _(jules)_ — Verify gain staging, LUFS targeting accuracy, limiter ceiling, and WAV export bit depth.

## Milestone 7: Presets System
Let users save and load named presets covering all vocal/instrumental/mastering control values.

  - **Implement Preset data model and persistence** _(claude)_ — app/models/preset.py dataclass plus save/load/list/delete functions backed by JSON files in the cache presets folder.
  - **Test preset round-tripping** _(jules)_ — Verify save/load/list/delete correctness and error handling for missing presets.

## Milestone 8: Background Job Orchestration & First-Run Setup Wizard
Wire the full pipeline into a cancellable background job with staged progress, and build the guided first-run setup UI on top of the model downloader.

  - **Build the render job worker** _(claude)_ — app/workers/render_job.py: QThread/QRunnable orchestrating ingestion->separation->vocal/instrumental chains->remix/master, emitting stage progress and supporting cancellation.
  - **Build the first-run guided setup wizard UI** _(antigravity)_ — PySide6 wizard: welcome, CUDA detection result, model download with live progress bars per model, completion screen.
  - **Test job cancellation and failure paths** _(jules)_ — Verify RenderJob signal sequencing, cancellation mid-run, and error propagation using mocked pipeline stages.

## Milestone 9: Main Application UI Shell
Build the main window with the two sections (Stem Separation, Artifact Fixing & Mastering) and track loading flow.

  - **Build main window and navigation shell** _(antigravity)_ — PySide6 MainWindow with file load/drag-drop, and navigation between the Stem Separation section and the Artifact Fixing & Mastering section.
  - **Bug sweep on file load and navigation flow** _(jules)_ — Manually exercise file loading (drag-drop, picker, invalid files) and section navigation for crashes or dead-ends.

## Milestone 10: Vocal & Instrumental Control Panels
Build the detailed control panels for both stems: toggles, intensity sliders, EQ/notch controls, gain, and the render-on-apply button.

  - **Build vocal control panel** _(antigravity)_ — Denoise/Enhance toggles+intensity, Vocal Clean Intensity blend, Vocal Gain, Harshness Cut (4kHz notch depth), Apply/Render button, preset picker.
  - **Build instrumental control panel** _(antigravity)_ — Mirrored granular controls for the instrumental chain per the 'detailed_controls' decision.
  - **Test control panel state and signal emission** _(jules)_ — Verify slider/toggle state changes, preset load/save wiring, and correct Settings assembly on render trigger.

## Milestone 11: Waveform A/B Player & Render History UI
Build the Original vs Cleaned waveform players with scrub/seek, and the render history list backed by the persistent cache.

  - **Build waveform A/B player widget** _(antigravity)_ — Side-by-side waveform display for original and cleaned audio with scrub/seek and synced or independent playback.
  - **Build render history panel** _(antigravity)_ — List of recent cached renders for the current track with settings-used metadata, allowing the user to select and load one into the Cleaned player, plus a Clear Cache action.
  - **Test waveform rendering and render history behavior** _(jules)_ — Verify waveform data computation correctness and render history list/selection/clear-cache behavior.

## Milestone 12: Packaging & Installer
Build the PyInstaller executable and wrap it in an Inno Setup guided installer that launches the first-run setup wizard.

  - **Configure PyInstaller build and Inno Setup script** _(claude)_ — PyInstaller spec producing MusicMasteryEnhancer.exe, and an Inno Setup script producing a guided installer that runs the app and triggers first-run setup on first launch.
  - **Bug sweep on packaged build and first-run flow** _(jules)_ — Exercise the built .exe's first-run setup, CUDA detection on a GPU-less environment, and normal launch path for packaging-specific issues.

## Milestone 13: End-to-End Integration & QA
Run the full pipeline through the real UI on representative Suno tracks, verify GPU and CPU code paths, and sweep for remaining bugs before calling the app done.

  - **Full pipeline integration test** _(jules)_ — Automated test driving ingestion through export using real (short) audio fixtures on both mocked-GPU and forced-CPU paths.
  - **Cross-cutting bug and UX sweep** _(jules)_ — Manual/automated sweep across the full UI (load -> separate -> adjust controls -> render -> compare -> export) for crashes, dead ends, or spec deviations.
