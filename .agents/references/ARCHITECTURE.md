# Architecture

> Generated by Sauron from the discovery interview. Treat as the source of truth for this project.

## Stack
- Python 3.11
- PySide6 (Qt6) for the native desktop UI
- audio-separator with a BS-RoFormer checkpoint (e.g. model_bs_roformer_ep_317_sdr_12.9755.ckpt) for vocal/instrumental stem separation
- resemble-enhance for neural denoise + harmonic reconstruction on both stems
- Pedalboard (Spotify) for the DSP chain: HPF/LPF, peak EQ notch, sidechain-compression-based de-esser, and the final mastering limiter
- pyloudnorm for LUFS loudness measurement and normalization targeting
- soundfile + ffmpeg (bundled binary via imageio-ffmpeg) for WAV/MP3 I/O and format conversion to 44.1kHz/24-bit WAV
- PyTorch as the ML backend, CUDA-optional with automatic CPU fallback
- Custom QPainter-based (or pyqtgraph) waveform widget for the A/B player scrub/seek UI
- QThread/QRunnable + Qt signals for cancellable background render jobs with per-stage progress
- JSON files for presets and render-history metadata
- PyInstaller to build the .exe, Inno Setup to build the guided installer
- pytest + pytest-qt for automated testing

## Key Decisions
- Neural passes (audio-separator, resemble-enhance) are cached per stem keyed by input-file hash + neural settings, so DSP-only re-renders (EQ, blend, gain, de-ess) skip the slow neural step — this is what makes render-on-apply feel responsive
- GPU is optional: torch.cuda.is_available() is checked at startup and before each render; CPU fallback is always available so the app works on any Windows machine, just slower
- Model weights are NOT bundled in the installer; they are downloaded during a guided first-run setup wizard with progress + checksum verification, keeping the installer small
- Render jobs run on a background QThread with per-stage progress signals (Separating/Denoising/Mixing/Mastering) and a cancel token, so the UI thread never blocks
- Stems, renders, and their metadata persist in a local cache (%LOCALAPPDATA%\MusicMasteryEnhancer\cache) across sessions rather than temp files, enabling the 'keep recent renders' comparison feature and a manual Clear Cache action
- Vocal and instrumental each get their own neural+DSP chain with full granular controls (mirrored UI), because Suno artifacts appear in both stems, not just vocals
- The final mastering stage (true-peak limiter + LUFS normalization, default -14 LUFS) runs after the vocal+instrumental blend, as a distinct pipeline stage from the per-stem cleanup chains
- Pipeline logic lives entirely in app/core, decoupled from Qt — UI widgets only dispatch to worker wrappers and render results, never contain DSP/ML logic, so core logic is independently testable
