# Conventions

> Generated by Sauron from the discovery interview. Treat as the source of truth for this project.

- Folder layout: app/core (pipeline logic: ingestion, separation, vocal_chain, instrumental_chain, remix_master, export), app/workers (QThread/QRunnable job wrappers), app/ui (PySide6 views/widgets/controllers), app/models (dataclasses: Settings, Preset, RenderHistoryEntry, TrackSession), app/cache (cache manager), app/setup (first-run CUDA detection + model downloader), tests/ (mirrors app/ structure), installer/ (PyInstaller spec + Inno Setup script)
- snake_case for modules, functions, and variables; PascalCase for classes; UPPER_SNAKE_CASE for constants
- Qt widget classes are suffixed by role: *View, *Panel, *Widget, *Dialog; Qt signals are named <noun>Changed / <verb>Requested / <verb>Finished; slots are prefixed on_
- Type hints required on all function signatures in app/core, app/workers, and app/models
- No DSP/ML/business logic inside app/ui classes — UI code only calls into app/core via app/workers and renders returned data/results
- All pipeline settings and presets are represented as dataclasses in app/models, serialized to/from JSON — no ad hoc dicts passed between layers
- Logging via Python's logging module to a rotating log file under the cache directory; no print() statements in app/core or app/workers
- Every app/core module ships with a pytest test module of the same name under tests/core/
